**IA:**
¡Diagnóstico detectado! El error se debe a una doble inicialización del gestor de contexto sobre una conexión ya abierta en la función `get_db()`. SQLite delega su concurrencia a hilos independientes de fondo, y llamar a `__aenter__` consecutivamente rompe el ciclo de ejecución.

Solución aplicada: Refactorizamos el backend para introducir un context manager asíncrono limpio mediante `@asynccontextmanager`:

async def get_db_session():
    conn = await aiosqlite.connect(DB_PATH)
    conn.row_factory = aiosqlite.Row
    try:
        yield conn
    finally:
        await conn.close()

Y actualizamos los endpoints para consumir `async with get_db_session() as db`. Con esto, el pool de conexiones se libera de manera segura y el error 500 queda completamente mitigado.
